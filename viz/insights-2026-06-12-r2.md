- **gpt-5.5 tops quality (21/25)**; opus-4-6, opus-4-8, and all three
  Fable 5 efforts tie at 20/25. The quality ceiling is real but flat:
  five configs within one task of each other.

  per correct**, 3× cheaper than gpt-5.5 ($0.0119) and 10× cheaper
  than Fable 5 (~$0.040+) while giving up 2–3 tasks.
- **The benchmark rerun fired a pre-authorized scorer trigger**: both
  parallel tasks systematically under-credited correct topical
  fan-outs that scoped the counterparty via a metadata `filters` key
  instead of a query token. PR #25 (per-spec alternatives) shipped
  mid-distillation; this snapshot re-scores all 12 configs uniformly
  from recorded calls. 9 of 12 configs were rescued on at least one
  parallel task; zero regressions; title-guess filters stay 0.
- **Axis E saturated on arrival** (12/12) and is reclassified as
  baseline floor. The adversarial reviewer on PR #22 predicted exactly
  this; the discriminator the axis bet on (colloquial period phrase →
  exact boundary) is below the current frontier across all tiers.
- **The decompose split reproduced exactly**: `vendor_renewal_decompose_01`
  again passes only on sonnet-4-6 and opus-4-6 (2/12, same two passers
  as the morning run) — strong evidence decompose-on-cue is a 4.6-era
  stylistic reflex, not a capability gradient.

## 4. Behavioral findings

### Distractor susceptibility — still all `summarize_document`

8 distractor picks total, all `summarize_document`, concentrated on
sonnet-4-6 (6 picks: 4× on `halverson_dispute_03`, 1× each on
`_04` and `stonebridge_term_01`); gemma and gpt-4o-mini take 1 each
on `stonebridge_term_01`. The near-miss shadow tool works; the three
categorically-wrong distractors and `search_history` get zero traffic
from anyone.

### Decompose vs. parallel — the split is a fingerprint

`vendor_renewal_decompose_01`: 2/12, **identical passer set to the
morning run** (sonnet-4-6, opus-4-6). Meanwhile 9/12 configs execute
correct parallel fan-outs on the tri task. The newest models
(opus-4-8, Fable, gpt-5.5) skip explicit decomposition and go
straight to retrieval — consistent with decompose-on-cue being
RL'd out of newer assistants rather than a capability the ladder
climbs. Keep-strict was held for this run (decision: revisit now that
the rerun data is in; see §6).

### Effort-knob economics (Fable 5)

Identical 20/25 at low, medium, and high; cost rises monotonically
($0.806 → $0.845 → $0.882), so CPC strictly worsens with effort.
The knob buys tokens, not retrieval strategy — same as June-10
(then 10/15 at all three efforts). One nuance: effort is not fully
behavior-neutral — Fable low fails `easton_amendment_03b` while
medium/high pass it (low instead passes `vendor_autorenew_02`).
The net is identical, the path differs.

### The 4.6 generation has a distinct profile

sonnet-4-6 and opus-4-6 are the only decompose-on-cue models AND
sonnet is the heaviest `summarize_document` consumer. The 4.6 pair
reads as tuned toward orchestration-style tool use; the 4.8/5.x
generation is tuned toward direct retrieval.

  it fails almost everything requiring state tracking.

## 7. Caveats & threats to validity

- Single run per config; n_tasks=25; n_configs=12. Report structural
  findings (frontier shape, behavioral splits, axis postmortems), not
  1-task gaps between adjacent configs.
- Parallel-task scores are offline re-scores under #25 from recorded
  `actual_calls`; single-call tasks keep runtime scores (their
  scoring rules did not change). `score_at_runtime` is preserved
  per config in the snapshot for auditability.
- The keyword-contains matcher (design decision 5) retains inherent
  leniency (substring topics); per-spec alternatives inherit it.
- Latency comparisons are coarse (mean over 25 sequential calls, one
  machine, one afternoon).
